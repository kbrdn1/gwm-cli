# [0.5.0-rc.1] - 2026-05-19

Delta against v0.4.0 stable. Merged PRs: #60, #61, #62, #63.

### Added

- `[doctor].trunks` in `.gwm.toml` (#61, closes #59) ‚Äî `gwm doctor`'s orphan-branch check now reads its trunk list from a new `[doctor]` table in `.gwm.toml`, default `["dev", "main"]`. Repos with non-standard trunk conventions (`master`, `trunk`, release trains like `release-3.x`) can opt in via config instead of seeing every merged gwm-style branch wrongly flagged as "unmerged orphan." An explicit empty list (`trunks = []`) disables the merge filter entirely. Zero-diff for repos without a `[doctor]` section.
- `.githooks/pre-commit` (#63, closes #56) ‚Äî opt-in POSIX hook that gates commits on two checks contributors previously enforced by hand. Gate 1 re-runs the test suite under a stripped PATH when staged `tests/*.rs` hunks touch ambient state. Gate 2 runs `gwm doctor` when staged paths touch `.gwm.toml`, `src/{bootstrap,doctor}.rs`, or `examples/gwm.toml.example`. Install with `git config core.hooksPath .githooks`.

### CI

- New `hook-smoke` job in `.github/workflows/ci.yml` (#63) ‚Äî cheap parallel job (~30 s, no Rust toolchain) that `shellcheck`s the pre-commit hook, asserts it is executable, verifies the empty-index short-circuit, and exercises the gate-2 "gwm not in PATH" skip path.

### Tests

- Sentinel / regression-only audit (#60, closes #57) ‚Äî classified all 193 tests at v0.4.0 into three buckets (drives logic / regression sentinel / dead weight), deleted 3 dead-weight tests, annotated 11 sentinels with a `// regression: ` tag. Suite: 193 ‚Üí 190. Full table in [`claudedocs/test-audit-0.4.0.md`](../../claudedocs/test-audit-0.4.0.md).
- `tests/config_tests.rs` ‚Äî 4 new cases covering the four `[doctor]` section states (absent, default, custom, empty) (#61).
- `tests/doctor_tests.rs` ‚Äî 3 new cases for trunk-config wiring: configured trunks honoured, empty trunks disables filter, missing trunk tolerated (#61).
- `tests/precommit_hook_tests.rs` ‚Äî 15 new integration tests stubbing `cargo` and `gwm` to drive hook gate behaviour deterministically (#63).
- `tests/cli_binary.rs` ‚Äî 4 new pinning tests for the `gcd` no-arg routing docs (#62).

### Docs

- `README.md` + `gwm shell-init` output + `gwm switch --help` ‚Äî document the `gcd` two-paths routing (#62, closes #58). The bare `gcd` (no argument) routes to `gwm switch` (interactive picker); `gcd <pattern>` routes to `gwm cd <pattern>` (fuzzy resolve). Bridge was wired in PR #53 but was undiscoverable; now surfaced consistently across all three doc surfaces.
- `examples/gwm.toml.example` ‚Äî documents the new `[doctor].trunks` knob with a commented-out block (#61).
- `CONTRIBUTING.md` ‚Äî new "Local hooks (recommended, opt-in)" subsection under Development (#63).
- `claudedocs/test-audit-0.4.0.md` ‚Äî per-file test classification table for the v0.4.0 suite, with deletion rationale and a forward-looking reading guide (#60).
