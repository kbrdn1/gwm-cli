# [0.5.0-rc.2] - 2026-05-19

Second release candidate for 0.5.0, picking up two additional feature merges since `v0.5.0-rc.1`.

### Added since rc.1

- **`gwm tmux <pattern>` + `gwm zellij <pattern>`** (#23, PR #65) — open the matched worktree in a new multiplexer window/tab. tmux opens a new window via `tmux new-window -n <name> -c <path>`; zellij opens a new tab via `zellij action new-tab --name <name> --cwd <path>`. Both verbs accept `-p` / `--split` to spawn into a split of the current pane instead (`tmux split-window` / `zellij action new-pane`). Detection is gated on `$TMUX` / `$ZELLIJ` — outside a multiplexer session the command refuses with a clear error rather than spawning a stray server. Zellij `--cwd` requires ≥ 0.40. New `src/multiplexer.rs` keeps the argv builders pure so `tests/multiplexer_tests.rs` can pin the incantations without requiring either binary on the runner.
- **TUI confirm-overlay safety countdown** (#30, PR #66) — when `delete branch on remove` (`p`) is armed, the `d → y` flow now requires a configurable countdown (default 3s, visualised by a 10-cell progress bar) before the destructive action fires. `Esc` / `n` cancels at any time; pressing `y` again during the countdown disarms it. New `[tui].confirm_countdown_secs` knob in `.gwm.toml` (range `0..=5`, default `3`); `0` falls back to the classic single-keystroke confirm; values above `5` are clamped on read so a typo can't strand the destructive path. The classic flow (`p` OFF) is unchanged — countdown only adds friction to the high-risk path the user has explicitly armed. State machine pinned by 12 `tests/tui_app_tests.rs` tests (`Instant` injected so the suite never sleeps); config parsing / clamping by 5 `tests/config_tests.rs` tests.

### Cumulative content

Everything from [`v0.5.0-rc.1`](0.5.0-rc.1.md) is included: pre-commit hook (#56), `[doctor].trunks` config (#59), `gcd` two-paths routing docs (#58), sentinel test audit (#57).

### Purpose

Validates two more flagship features for 0.5.0 — the multiplexer integration (#23) and the destructive-action safety net (#30) — before the stable cut.
